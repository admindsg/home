# Homepage Content Requirements

## Hero copy

Required elements:

- Tagline: “Talent is everywhere. Opportunity starts here.”
- Identity: NYC-based 501(c)(3) nonprofit music organization.
- Model: helps people learn, record, and perform.
- Core promise: talent should never be limited by circumstance.
- Primary CTA: Find Your Path.
- Secondary CTA: Sponsor Opportunity or Donate.

Recommended direction:

> Discovery Sound Garden helps people learn, record, and perform through opportunity-centered music education, mobile recording support, and community performance pathways.

## Audience path copy

Each path needs:

- Audience label in plain language.
- One empathetic sentence.
- One proof/program connection.
- Primary CTA and secondary CTA.

Needed paths:

- “I want to start learning.”
- “I used to make music and want to return.”
- “I want to sing with others.”
- “I want to create or record.”
- “I want to support opportunity.”
- “I want to partner with DSG.”
- “I want to help the community.”

## Program copy

Program entries should include:

- Program name.
- Who it is for.
- What happens.
- What participants build or experience.
- Current status.
- CTA.

Program copy should cover:

- Foundations of Musicianship.
- Workshops.
- Vanguard Voices.
- Mobile recording opportunity.
- Partner programming.

## Opportunity-gap copy

Preserve the live-site argument:

- Cost keeps lessons, studio time, and coaching out of reach.
- Opportunity is unevenly distributed by school, neighborhood, and network.
- Gatekeeping means opportunities often flow through connections rather than talent.
- DSG builds the missing bridge between structure, recording opportunity, and public outcomes.

Visible copy should prefer “opportunity” over “access” unless discussing technical accessibility or disability.

## Trust proof

Needed proof types:

- 501(c)(3) status.
- NYC community-centered identity.
- Program names and pathways.
- Vanguard Voices as public proof.
- Real photos/video when approved.
- Founder/leadership credibility.
- Impact metrics only if verified.
- Partner examples only if approved.

## Donor proof

Needed donor content:

- Donation impact levels: $25, $50, $100, Custom.
- Clear Zeffy donation link.
- Nonprofit reassurance.
- Statement that gifts support learning, recording opportunity, materials, sponsored seats, community programming, and the mobile studio pathway.
- No inflated impact claims.

## Partner proof

Needed partner content:

- What DSG can bring: workshops, residencies, mobile recording, showcases, community activations.
- Who DSG can serve: schools, community organizations, arts partners, youth/adult learners.
- What outcomes partners can expect: learning, confidence, creative proof, performance/community outcomes.
- Contact/inquiry workflow.

## Image needs

Final website photos should not be stored in GitHub. Use Cloudflare Images:

`https://imagedelivery.net/SRt55pBu4n6Lz33qNPNE1g/<image_id>/<variant_name>`

Needed images:

- Hero/community music image.
- Vanguard Voices rehearsal or performance image.
- Learning/workshop image.
- Recording/mobile studio image.
- Founder/leadership image if approved.

If final images are not ready, use polished non-image content blocks or abstract CSS treatment. Do not render placeholder labels.

## Video needs, if relevant

Video is optional for the next build. If used later:

- Use only approved DSG video.
- Provide captions or transcript.
- Do not autoplay with sound.
- Lazy-load embeds.
- Provide fallback link.

## Real testimonial policy

- Do not invent testimonials.
- Use quotes only with permission.
- If quoting participants, define attribution policy: full name, first name, initials, or role-based attribution.
- Do not imply a stock/AI person is a DSG participant.
- If testimonials are not ready, use program proof and founder-approved impact statements instead.

## Language to avoid

Avoid in visible UI:

- “Sound Garden Field Guide.”
- “Field note.”
- “Specimen.”
- “Seed → Roots → Stem → Bloom” as pathway labels.
- “Image pending.”
- “Cloudflare pending.”
- “TODO.”
- “Coming Soon” as a dead end.
- Generic phrases like “unlock your potential” unless tied to DSG-specific outcomes.
- Excessive “access” language where “opportunity” works better.

## Internal TODO language that must never appear in rendered UI

Allowed only in docs, comments, or issue/task descriptions:

- TODO
- placeholder
- image pending
- Cloudflare pending
- final asset pending
- analytics pending
- founder review needed
- replace with final copy

- Partner inquiry destination.

- Approved Cloudflare Image IDs and variants.

- Approved testimonials or decision to launch without testimonials.

- Whether volunteer/community ally has a direct public pathway.
