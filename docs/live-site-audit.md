# Live Site Audit: Discovery Sound Garden Homepage

Sources reviewed on 2026-07-04:

- Current live website: <https://discoverysoundgarden.com/>
- Experimental GitHub Pages preview: <https://admindsg.github.io/home/>
- Repository homepage preview files: `index.html`, `assets/css/styles.css`, `assets/js/site-data.js`

## Summary of the current live website

The current live Discovery Sound Garden website is more persuasive than the experimental preview because it already explains the nonprofit's core thesis: NYC has talent, but opportunity is uneven, expensive, and connection-driven. The homepage connects that problem to a clear organizational model: music education, mobile recording, and performance pathways.

The live site's strongest strategic asset is its language. It already contains several sentences that feel specific to DSG and should anchor the next homepage rebuild:

- “Talent is everywhere. Opportunity starts here.”
- “The problem isn't talent. It's opportunity.”
- “Music Education • Mobile Recording • Performance Pathway”
- “DSG builds the missing bridge: structure + recording access + public outcomes.”
- “Build the skill. Record the proof. Let the world hear it.”
- “Discovery Sound Garden cultivates possibility. Vanguard Voices lets the world hear what it becomes.”

The live site's biggest weakness is not mission clarity; it is conversion architecture. It has many good ingredients, but the page does not yet guide different audiences into tailored next steps. Beginners, returning adults, community singers, emerging artists, donors, schools, and partners are present, but they are not sequenced into a mobile-first journey that quickly answers “Is this for me?” and “What should I do next?”

## Strongest current messages

### Preserve as homepage anchors

1. **Required tagline**  
   “Talent is everywhere. Opportunity starts here.” This should remain the hero headline or the visible hero tagline.

2. **Core problem statement**  
   “The problem isn't talent. It's opportunity.” This is specific, memorable, and directly supports the nonprofit case for giving, partnering, and participating.

3. **Core pathway**  
   “Music Education • Mobile Recording • Performance Pathway” and “Learn → Record → Perform” should become the structural spine of the homepage.

4. **Outcome language**  
   The live site is strongest when it describes proof: songs recorded, performances produced, confidence built, and public outcomes.

5. **Bridge language**  
   “Structure + recording access + public outcomes” is strategically useful, though visible copy should prefer “recording opportunity” over “recording access” where possible.

6. **Vanguard Voices as proof**  
   Vanguard Voices should not be presented as just another program card. It should be the public proof module showing that DSG's pathway can become audible, visible community excellence.

## Strongest current page sections

1. **Hero and mission opening**  
   Strong because it names the pathway and the promise quickly.

2. **Opportunity gap explainer**  
   Strong because it gives donors and partners a reason to care beyond generic arts education language.

3. **Products & services / model explanation**  
   Strong because it distinguishes DSG from a lesson studio, a chorus, and a recording service by linking those elements into one pathway.

4. **Audience path section**  
   Strong conceptually because it acknowledges different entry points: students, emerging artists, adults returning to music, and schools/partners. It needs interaction design and clearer CTAs.

5. **What's growing now**  
   Useful because it introduces active and upcoming programs, but it needs more specificity, dates, current availability, and founder-approved status.

6. **Donation impact levels**  
   Strong because the amounts are concrete and connected to participant outcomes.

## Existing audience paths

The live site already implies these paths:

- **Students / beginners:** “Build your foundation,” workshops, Foundations of Musicianship.
- **Emerging artists:** affordable demos, production support, visibility without gatekeeping.
- **Adults returning to music:** supportive re-entry into musicianship, confidence, and community.
- **Schools and partners:** workshops, residencies, mobile recording, showcases, community activations.
- **Community singers:** Vanguard Voices as a chorus/performance pathway.
- **Donors:** sponsor workshop seats, learning materials, demos, community programming, mobile studio pathway.

Missing path clarity:

- There is no fast “choose your path” interaction for mobile users.
- Community singers are partly hidden under Vanguard Voices rather than named as an audience path.
- Volunteers/community allies are not clearly represented.
- Partners need a direct inquiry CTA and clearer examples of what DSG can bring to a school or organization.

## Existing donation language

Current donation language worth preserving or adapting:

- “Help turn potential into proof.”
- “Every gift directly sponsors...”
- “$25 sponsors access to a workshop for one participant.”
- “$50 supports a participant's learning materials or recording access.”
- “$100 helps fund supported seats, demos, and community programming.”
- “Custom: Help build the mobile studio pathway. Every dollar moves the mission forward.”

Recommended rewrite direction:

- Replace visible “access” with “opportunity” unless the sentence is intentionally about technical accessibility or disability.
- Keep concrete giving levels, but turn them into an interactive donor impact selector on the next homepage.
- Add reassurance that Zeffy is the donation processor and donations support DSG's nonprofit mission.

## Existing program language

Program language to preserve:

- **Music Education:** lessons, workshops, Foundations of Musicianship, rhythm, theory, ear training, composition, voice, instruments, production.
- **Mobile Studio / Recording:** portable recording model, songwriting, demos, production, content capture, schools, communities, emerging artists.
- **Vanguard Voices:** flagship chorus, musicianship, blend, leadership, community performance, public excellence.
- **Foundations of Musicianship:** 8-session structured path; rhythm, melody, harmony, confidence.

Program language that needs clarification:

- “Our Products & Services” sounds commercial and should be rewritten as “Programs,” “Pathways,” or “What DSG Builds.”
- “Recording access” should become “recording opportunity” or “mobile recording opportunity.”
- “Coming Soon” is acceptable only when contextualized in non-public internal docs or a carefully designed roadmap; avoid isolated dead-end labels in the public UI.
- “Starting September” needs a specific year or should be removed until confirmed.

## Existing trust signals

- NYC-based nonprofit identity.
- 501(c)(3) status.
- Specific mission and vision statements.
- Concrete program names: Vanguard Voices and Foundations of Musicianship.
- Concrete donor impact amounts.
- Visible external donation path through Zeffy.
- Instagram/social presence.
- Public outcome framing: recordings, performances, confidence, community.

Additional trust signals needed:

- Founder/leadership name and short credibility statement.
- Real photos with permissions.
- Real program dates or “join notifications” if dates are not final.
- Partner examples or categories.
- Impact numbers only if verified.
- Real testimonials only if approved and attributable or intentionally anonymized with consent.

## Navigation issues

- The live site includes multiple top-level links, but the homepage journey still feels more like a set of sections than a guided decision flow.
- “Program Offerings,” “Why Our Sound Garden,” “About DSG,” “Donate to DSG,” and “Vanguard Voices” are reasonable, but mobile visitors need clearer action labels.
- The next build should prioritize 3–5 primary actions: Find Your Path, Programs, Vanguard Voices, Partner, Donate.
- Donation should remain persistent but not overpower participant paths.
- If a mobile sticky action bar is used, it should contain high-intent actions only, such as Find Your Path and Donate, with a third “Stay Updated” action only if testing shows value.

## CTA issues

- CTAs sometimes describe destinations rather than outcomes.
- Some CTAs lead to “Coming Soon” states, which can break momentum.
- Emerging artists and partners need more explicit action options.
- Donors need an impact explanation before the Zeffy jump.
- Community singers need a direct Vanguard Voices CTA.

Recommended CTA system:

- Primary participant CTA: “Find Your Path.”
- Education CTA: “Start with Foundations.”
- Singer CTA: “Explore Vanguard Voices.”
- Artist CTA: “Explore Recording Opportunity” or “Get Recording Updates” until the program is active.
- Partner CTA: “Start a Partner Conversation.”
- Donor CTA: “Sponsor Opportunity.”
- Newsletter/event CTA: “Get Event Notifications.”

## Mobile issues and opportunities

Observed/strategic mobile issues:

- Current messaging is strong but too much of the decision-making is linear and text-heavy.
- The homepage needs a fast path selector near the top because mobile visitors are often scanning in short sessions.
- Cards should not become long undifferentiated stacks without memory of the user's selected path.
- Donation levels should be tappable and readable without tiny controls.
- Embedded Zeffy content needs a fallback link because iframes can fail, load slowly, or feel cramped on phones.
- Navigation should not require users to understand DSG's full model before acting.

Mobile-first opportunities:

- A “Find Your Path” selector with large buttons and path-specific panels.
- A compact Learn → Record → Perform section that can expand into details.
- Accordions for programs, not hidden information that blocks SEO.
- A donor impact selector that updates visible impact copy without requiring page navigation.
- A persistent but respectful action area after the hero or as a bottom bar, tested for overlap and cognitive load.

## SEO opportunities

Current SEO strengths:

- The live title includes “Music Education & Recording Access NYC,” which targets location and service intent.
- Core terms include music education, recording, performance, nonprofit, NYC, workshops, chorus, and mobile recording.

SEO improvements for the next build:

- Use one clear H1 with the tagline.
- Add descriptive H2 sections: “Music Education, Recording Opportunity, and Performance Pathways in NYC,” “The Opportunity Gap,” “Learn → Record → Perform,” “Vanguard Voices,” “Partner with DSG,” and “Sponsor Opportunity.”
- Add meta description that includes NYC, nonprofit music organization, learn, record, perform.
- Add Open Graph/Twitter meta tags with approved imagery once available.
- Use schema only if maintainable: Organization and NonprofitOrganization structured data can be added later with verified address/social details.
- Avoid hidden, duplicate, or placeholder copy.
- Ensure link text is descriptive and not only “Click Here.”

## Where the current live site persuades well

- It names a real problem in plain language.
- It positions DSG as a bridge rather than a generic lesson provider.
- It links learning to outcomes, which matters for both participants and donors.
- It gives donors concrete impact levels.
- It makes Vanguard Voices feel like proof of the thesis.

## Where the current live site loses clarity or momentum

- Audience paths are present but not interactive or strongly sequenced.
- Program cards do not always clarify current availability.
- “Access” appears often enough to dilute the preferred “opportunity” language.
- Some action links are generic or lead to unclear readiness states.
- Newsletter/social embeds create “Loading...” text that can feel broken if slow.
- The homepage does not yet create a strong mobile conversion path.

## Recommended content to preserve

- Tagline and core promise.
- Opportunity gap language.
- Learn → Record → Perform pathway.
- Build the skill / Record the proof / Let the world hear it.
- Vanguard Voices as public proof.
- Foundations of Musicianship as a structured entry point.
- Donor impact amounts and outcome-based giving logic.
- Audience segmentation for students, adults, emerging artists, schools, partners, donors.

## Recommended content to rewrite

- “Products & Services” → “Programs and Pathways” or “How DSG Builds Opportunity.”
- “Recording access” → “Recording opportunity” in visible marketing copy.
- “Click Here” → action-specific CTA.
- “Coming Soon” → “Get updates,” “Pilot details coming after founder approval,” or a non-visible internal note.
- General “accessible/affordable” claims → more specific opportunity-centered claims.

## Recommended content to remove or avoid

- Public-facing placeholder language such as “image pending,” “Cloudflare pending,” “final approved image pending,” or “TODO.”
- Literal field-guide labels such as “field note,” “specimen,” “Sound Garden Field Guide,” and “Seed → Roots → Stem → Bloom” unless explicitly approved later.
- Generic nonprofit card grids with interchangeable mission language.
- Fake testimonials, fabricated impact numbers, or stock imagery used as if it depicts DSG.
- SaaS-style conversion language that makes DSG feel like a product funnel rather than a community organization.

## Experimental GitHub Pages preview audit

### Useful technical structure

- Plain HTML, CSS, and minimal JavaScript are aligned with the future build constraints.
- Centralized site data in `assets/js/site-data.js` is the right direction for links, embeds, Cloudflare Images URLs, and future analytics placeholders.
- Skip link, semantic sections, and deferred scripts are useful.
- The existing section order can be used as a rough skeleton: hero, paths, problem, pathway, programs, proof, donation, connect.

### Design direction that failed

- The “Sound Garden Field Guide” concept is too literal and makes the site feel like a theme rather than a credible nonprofit music organization.
- “Seed / Roots / Stem / Bloom” over-explains the garden metaphor and competes with the stronger Learn → Record → Perform pathway.
- “Specimens,” “field notes,” and similar labels feel decorative and internally clever instead of participant-centered.
- The preview underbuilds persuasion: it names sections but does not provide enough proof, audience guidance, program specificity, or conversion confidence.

### Public-facing placeholder language to remove

- “Placeholder for a future approved Vanguard Voices performance image.”
- “Cloudflare Images placeholder — final approved image pending.”
- Any alt text that announces placeholder status.
- Any rendered “TODO,” “pending,” or implementation commentary.

### Missing interactions

- No functional path selector that changes content based on audience.
- No accordion or comparison model for programs.
- No donor impact selector beyond static list text.
- No robust mobile navigation behavior.
- No embed fallback state beyond links.
- No future analytics event plan.

### Sections that are too thin

- Hero: strong tagline, but concept label weakens credibility.
- Find Your Starting Point: useful, but each path needs a panel with proof and next step.
- Opportunity Gap: directionally right, but should reuse stronger live-site specificity about cost, uneven opportunity, and gatekeeping.
- Program areas: too short to help users choose.
- Vanguard Voices: too thin without real proof, image, performance context, or join path.
- Sponsor Opportunity: useful levels, but needs impact selector behavior and trust reassurance.
- Stay Connected: should be less garden-themed and more practical.

### Reuse safely

- Static architecture and no-build approach.
- Centralized link/data approach.
- Skip link and semantic landmarks.
- Overall idea of path selection near the top.
- Basic section sequence after removing field-guide metaphor.

### Discard

- Literal field guide concept.
- Seed/Roots/Stem/Bloom as public labels.
- Placeholder image language.
- Underdeveloped program cards.
- Any generic or decorative labels that hide the concrete DSG promise.
